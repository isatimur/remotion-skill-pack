Spring Transitions

Every scene pair is connected via @remotion/transitions — organic spring physics, not linear fades. Declare per scene with transitionIn.

{
  "id": "s2-insight",
  "type": "insight",
  "durationFrames": 150,
  "transitionIn": "slide",   // fade | slide | wipe | flip | none
  "props": {
    "claim": "The bottleneck has shifted from compute to taste.",
    "category": "Key Insight",
    "source": "Chapter 2"
  }
}

Generated TSX requires @remotion/transitions: npm install @remotion/transitions

6 MCP Tools

Connect the MCP server to your AI assistant for interactive composition authoring. The server exposes a stdio protocol compatible with Claude Code, Cursor, Copilot, Codex, and Gemini CLI.

create_composition Generate a starter spec from title, theme, format, and key ideas. Returns landscape or social (9:16) structure.
validate_composition Validate a composition JSON against the schema. Returns errors with paths or { valid: true }.
render_to_tsx Convert a composition JSON to a ready-to-run Scene.tsx string with TransitionSeries wiring.
list_themes List all 3 themes with palette summaries (background, accent, dimensions, animation config).
get_scene_template Return example props + durationFrames for any of the 14 scene types — including all 6 new knowledge video types.
generate_composition_from_textnew Analyze raw prose (blog post, book chapter, transcript) — detects stats, comparisons, code blocks, and named people — and returns a full composition blueprint with per-scene guidance notes. Use this first when converting text to video.
